❓ よくある質問
スキルはどんな形式で届きますか？▼
購入後すぐに .skill ファイル（ZIPアーカイブ）をダウンロードできます。ファイルをダブルクリックするだけでClaudeに装備できます。
どのAIに対応していますか？▼
主にClaude（Anthropic）に対応しています。一部スキルはChatGPT・Geminiでも使用可能です。各スキルの詳細ページに対応AIが記載されています。
返金はできますか？▼
購入後7日以内であれば、スキルが説明通りに動作しない場合に限り全額返金対応します。お問い合わせフォームからご連絡ください。
スキルを出品するには何が必要ですか？▼
SKILL.mdファイルを含むスキルディレクトリがあれば出品できます。審査では動作確認・説明文の確認を行います。メールでお問い合わせください。
支払い方法は何が使えますか？▼
クレジットカード（VISA・Mastercard・JCB等）、PayPal、コンビニ払いに対応予定です。正式オープン時に詳細をお知らせします。
コミュニティ機能は誰でも使えますか？▼
無料で会員登録するとコミュニティへの投稿、他メンバーのフォロー、タイムラインの閲覧ができます。購入なしでも参加可能です。
ベータ版で先行登録すると何かメリットはありますか？▼
先行登録された方には、正式オープン時に30%割引クーポンをプレゼントします。また、新着スキルの最速通知をお届けします。

⚙️ Supabase セットアップ

無料でコミュニティ機能を有効化する方法（5分でできます）

① Supabaseアカウントを作る
  1. supabase.com にアクセス
  2. 「Start your project」→ GitHubでサインイン
  3. 「New project」→ プロジェクト名を入力して作成
② テーブルを作る（SQLをコピペするだけ）

Supabaseダッシュボード → SQL Editor → New query に以下をペースト：

create table profiles (id uuid references auth.users(id) primary key, username text unique not null, display_name text, bio text, avatar_emoji text default '🧑', created_at timestamp with time zone default now());

create table follows (follower_id uuid references profiles(id) on delete cascade, following_id uuid references profiles(id) on delete cascade, created_at timestamp with time zone default now(), primary key (follower_id, following_id));

create table posts (id uuid default gen_random_uuid() primary key, user_id uuid references profiles(id) on delete cascade, content text not null, likes integer default 0, created_at timestamp with time zone default now());

alter table profiles enable row level security;
alter table follows enable row level security;
alter table posts enable row level security;

create policy "Public read" on profiles for select using (true);
create policy "Own update" on profiles for update using (auth.uid() = id);
create policy "Own insert" on profiles for insert with check (auth.uid() = id);
create policy "Public read" on follows for select using (true);
create policy "Own manage" on follows for all using (auth.uid() = follower_id);
create policy "Public read" on posts for select using (true);
create policy "Own insert" on posts for insert with check (auth.uid() = user_id);
create policy "Own delete" on posts for delete using (auth.uid() = user_id);
③ APIキーをこのファイルに貼る

Settings → API から「Project URL」と「anon key」をコピーして index.html の下記の場所に貼り付け：

const SUPABASE_URL = 'https://xxxxxxxx.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciO...';
